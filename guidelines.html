Black

Color Name & Hex Value

Primary color, --black, #000000

Context of Use

Sitewide

  • Site-nav logo text and icon outline
  • All footer text (span & p)
  • Site-nav notifications icon
  • All search icons

Home Page

  • main headings (h2)
  • main body text (p)

Support & FAQ Page

  • form labels and user input field text (label & input)
  • form text input field borders (input)
  • All accordion text (headings and body text, summary & p)
  • Accordion arrow icons

Search Page

  • Filters dropdown text (select & option)
  • Filters borders (select)
  • Post headings and body text (h3 & p)

Messages Page

  • Vertical nav links and page title (p & h1)
  • Convo-cards titles (p)
  • Select all label (p)

Single-Post Page

  • Page title (h1)
  • create-post user input field text and borders (textarea)
  • Create-post filter dropdown text & borders (input)
  • Add-more-tags tag text (p)
  • Save-as-draft button text (a)

Login Page

  • Page title and subtitle (h1 & p)
  • form user input field labels, text, and borders (label & input)

Rationale

In our design, black fulfills the function it commonly does in many sites. It’s the default color, particularly for large bodies of text. It’s also used to contrast between lighter colors to make everything easier to read and understand.

Rationale

Inter and Lato are popular fonts among web designers due to the readability and versatility of both the fonts. Both fonts are designed with readability in mind, making them easy to read on various digital device screens.


Inter and Lato can be used for a wide range of applications and contexts. We use Lato as a heading font, because its wide letter form and thick bold letters draws users’ attention. Inter is utilized as a body text font because of its contrasting thin lettering to Lato and frequent application within the web design community.

Grid-Based Layout System

Responsive Web Design

This layout system is designed based on a 12 column grid layout. Grid content scales to fit the area on the page, depending on what the component is. Some components scale to fit the page, and all components will fit on any given page, no matter how small it is. However, some components have max widths of how wide a given component can be, so ultra-wide screen viewers don’t have a completely different view of what the site should look like, and don’t have to read large blocks of text that span a larger computer screen.


For desktop viewers, there is a margin of 100px around the body pages of the design. Elements that are right next to eachother in the grid layout have a gutter of 10px. Not everything in the current prototype is responsive, but components should be able to scale. With the help of flexbox, all components that are viewed in rows on the desktop should be able to be viewed in a column on smaller screens.
